# 开发笔记

Spec Kit 是一个用于规范驱动开发的工具包. 它的核心是一组协调的提示, 模板, 脚本和 CLI/集成资产, 用于定义和交付面向 AI 编码代理的规范驱动工作流. 本文档面向修改 Spec Kit 本身的开发者, 提供关键项目文档和仓库组织的简要概览.

**核心项目文档:**

| 文档                                                       | 作用                                                                               |
| ---------------------------------------------------------- | ---------------------------------------------------------------------------------- |
| [README.md](README.md)                                     | Spec Kit 及其工作流的主要用户侧概览.                                               |
| [DEVELOPMENT.md](DEVELOPMENT.md)                           | 本文档.                                                                            |
| [spec-driven.md](spec-driven.md)                           | Spec Kit 支持的规范驱动开发工作流的端到端说明.                                      |
| [RELEASE-PROCESS.md](.github/workflows/RELEASE-PROCESS.md) | 发布工作流, 版本规则和变更日志生成流程.                                             |
| [docs/index.md](docs/index.md)                             | `docs/` 文档集的入口.                                                              |
| [CONTRIBUTING.md](CONTRIBUTING.md)                         | 贡献流程, 审查期望和必需的开发实践.                                                |
| [TESTING.md](TESTING.md)                                   | 验证策略和测试流程.                                                                |

**主要仓库组件:**

| 目录                | 作用                                                                   |
| ------------------- | ---------------------------------------------------------------------- |
| `templates/`        | 定义核心工作流行为和生成制品的提示资产和模板.                          |
| `scripts/`          | 工作流, 设置和仓库工具所使用的支持脚本.                                |
| `src/specify_cli/`  | `specify` CLI 的 Python 源代码, 包括代理特定资产.                      |
| `extensions/`       | 扩展相关的文档, 目录和支持资产.                                        |
| `presets/`          | 预设相关的文档, 目录和支持资产.                                        |
